# UseJunction Launch Video

20 seconds, 30 fps, 1920x1080 master and 1080x1920 vertical adaptation.

## Creative Spine

Many tools in, one accountable view out. The edit moves from fragmented developer workflows to a single operating picture, then lands on the promise: teams keep their preferred tools while leaders get one control plane.

## Vertical Adaptation

- Keep the same timing and edit points.
- Recompose rather than letterbox: tool fragments stack vertically, junction nodes form a diamond column, dashboard is shown as a centered portrait crop with the active metric card and selected developer detail preserved.
- Keep all text inside a 9% safe margin. The final URL stays centered in the lower third.

## Audio Versions

### Narration

"Your team uses every AI coding tool. But nobody sees the full picture. UseJunction gives engineering leaders one control plane across tools, models, cost, latency, and failures. Open source. Self-hostable. usejunction.com."

### Music + Text Only

No voice. Use the same picture edit and replace the narration bed with a restrained technical pulse, preserving the text beats and final hold.

## Production Notes

- Use the real UseJunction public dashboard demo recording for the 4.7-16.0s product reveal. No fabricated dashboard values or fake testimonials.
- The repository did not include separate supplied screen-recording files when this plan was created; the implementation records the existing product demo component locally and flags this substitution in the render manifest.
- Palette: `#111210`, `#202419`, `#08758a`, `#e5ec67`, `#c0682c`, `#fafaf7`.
- Typography: DM Sans for message text; JetBrains Mono for metric labels and tool/model identifiers.
- Avoid generic 3D objects, robot imagery, excessive glass, and competitor-specific UI treatments.
